# Plan: Trustworthy 70B Prod Eval

## Context

The citation guard overhaul (shipped 2026-05-31) proved that prior correctness numbers (~40%) were corrupted by a broken NLI judge. Guard is now reliable. Generation is now unblocked with DeepInfra Llama-3.3-70B as a fallback provider. The next real quality signal is a trustworthy end-to-end eval using the winning prod config.

**Blockers today:**
1. `answer_eval_full.py --provider groq` burns Groq's 100k TPD on generation — 20 items × ~8k tokens ≈ 160k, overshoots the free tier before judging even starts.
2. `judge.py` and `answer_eval_full._get_judge()` are hardcoded Groq with no fallback — quota error mid-run = silent crash, no partial results.

**Goal:** Add `--provider deepinfra` that routes generation AND judging through DeepInfra Llama-3.3-70B (no daily quota ceiling, ~$0.35/1M tokens). Run full 20-item held-out eval with the winning config, record honest numbers in PROGRESS.md.

## Phase 1 — Code Changes ⚡ PARALLEL (both tasks independent)

### Task 1A · `evals/judge.py` — DeepInfra fallback

Add after `_get_judge()`:

```python
_deepinfra_judge_llm = None

def _get_deepinfra_judge():
    global _deepinfra_judge_llm
    if _deepinfra_judge_llm is None:
        from langchain_openai import ChatOpenAI
        _deepinfra_judge_llm = ChatOpenAI(
            model=os.environ.get("DEEPINFRA_MODEL", "meta-llama/Llama-3.3-70B-Instruct"),
            openai_api_key=os.environ["DEEPINFRA_API_KEY"],
            openai_api_base="https://api.deepinfra.com/v1",
            temperature=0,
        )
    return _deepinfra_judge_llm

def _is_quota_error(exc: Exception) -> bool:
    msg = str(exc).lower()
    return any(k in msg for k in ("rate_limit", "rate limit", "429", "quota", "exceeded"))
```

Update `score_item()` — wrap the `.invoke()` call:
```python
try:
    resp = _get_judge().invoke(messages)
except Exception as e:
    if _is_quota_error(e) and os.environ.get("DEEPINFRA_API_KEY"):
        resp = _get_deepinfra_judge().invoke(messages)
    else:
        raise
```

### Task 1B · `evals/answer_eval_full.py` — `--provider deepinfra`

Add `_force_deepinfra_generation()` (sets `config.LLM_PRIMARY`, uses rag.py's existing provider chain):
```python
def _force_deepinfra_generation():
    """Route production chain through DeepInfra 70B (no daily quota)."""
    import config as _cfg
    _cfg.LLM_PRIMARY = "deepinfra"
```

Add `"deepinfra"` to `--provider` choices:
```python
ap.add_argument("--provider", choices=["gemini", "groq", "deepinfra", "local"], default="gemini", ...)
```

Add handler in `main()` after the `elif args.provider == "local":` block:
```python
elif args.provider == "deepinfra":
    _force_deepinfra_generation()
    from langchain_openai import ChatOpenAI
    _deepinfra = ChatOpenAI(
        model=os.environ.get("DEEPINFRA_MODEL", "meta-llama/Llama-3.3-70B-Instruct"),
        openai_api_key=os.environ["DEEPINFRA_API_KEY"],
        openai_api_base="https://api.deepinfra.com/v1",
        temperature=0,
    )
    global _judge
    _judge = _deepinfra
    import judge as _judge_mod
    _judge_mod._judge_llm = _deepinfra
```

This replaces the `_judge` singleton in both files with the same DeepInfra instance — consistent scoring throughout.
